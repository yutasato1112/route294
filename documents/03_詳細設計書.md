#### 2.1.3 staff_required デコレータ

```python
def staff_required(view_func):
    @wraps(view_func)
    @login_required(login_url='/login/')
    def wrapper(request, *args, **kwargs):
        if not request.user.is_staff:
            raise PermissionDenied("権限がありません")
        return view_func(request, *args, **kwargs)
    return wrapper
```

**allocate メソッド（メインループ）**

```
best_score = -∞
best_allocation = None

for strategy_id in 0..199:
    allocation = 空の割り当て初期化

    hk_order = _get_strategy_order(strategy_id)
    floor_order = _get_floor_order(strategy_id)

    for hk_id in hk_order:
        _allocate_hk(hk_id, floor_order)

    _fallback_allocation()  # 未割り当て部屋の処理
    _adjust_twin_balance()  # ツイン配分調整
    _allocate_eco_rooms()   # エコ部屋割り当て

    score = evaluate_solution()

    if score > best_score:
        best_score = score
        best_allocation = copy(allocation)

result = _renumber_hks(best_allocation)
return result
```

**_allocate_hk メソッド（個別割り当て）**

```
引数: hk_id, floor_order
処理:
1. 利用可能フロアの検索（空き部屋がありMAX_HK_PER_FLOOR未到達）
2. バス担当の場合: 4F以下のフロアを優先
3. 第1フロアでツイン部屋を優先割り当て
4. 第1フロアの残りクオータをシングルで充足
5. クオータ未達の場合: 第2フロアで継続
6. フロア間距離制約: 2フロア以内
```

**_fallback_allocation メソッド（未割り当て処理）**

```
for each 未割り当て部屋:
    スコアリング:
    - 既存フロア一致: +500
    - 2フロア以内: +500〜800
    - クオータ余裕: +10/部屋
    - ツイン不足: +5

    最高スコアのハウスキーパーに割り当て
```

**_adjust_twin_balance メソッド（ツイン調整）**

```
Phase 1: 固定クオータ調整（50回反復）
    for each ハウスキーパーペア:
        if HK_A のツイン超過 and HK_B のツイン不足:
            HK_A のツイン1室 ←→ HK_B のシングル1室 をスワップ
            フロア制約チェック（2フロア以内）
            成功ならスワップ確定

Phase 2: 全体バランス調整（150回反復）
    max_twin_hk = ツイン最多のHK
    min_twin_hk = ツイン最少のHK

    if max - min > 1:
        直接スワップ or 3者チェーンスワップ を試行
        フロア制約チェック
```

**_allocate_eco_rooms メソッド（エコ割り当て）**

```
割り当て優先順位:
1. エコアウト部屋: 該当フロアに既存割り当てがあるHK
2. 通常エコ: 既存フロア + 平均以下のエコ数
3. 通常エコ: 2フロア未満 + フロア追加可能
4. 通常エコ: 3rdフロア追加可能（エコ専用）
5. 通常エコ: 既存フロア（エコ数不問）
6. 通常エコ: 3rdフロア追加（フォールバック）
7. 通常エコ: 任意のHK（最終手段）
```

**evaluate_solution メソッド（評価関数）**

```
score = 0

# 部屋クオータ精度
for each HK:
    normal_count = 通常部屋数
    room_diff = |normal_count - room_quota|
    score -= room_diff * 10,000

# フロア制約
    normal_floors = 通常部屋のフロア数
    score -= max(0, normal_floors - 2) * 5,000
    all_floors = 全部屋のフロア数
    score -= max(0, all_floors - 3) * 3,000
    if all_floors == 3: score -= 2,000

# ツインクオータ
    twin_count = ツイン部屋数
    twin_diff = |twin_count - twin_quota|
    score -= twin_diff * 100

# エコバランス
eco_diff = max(eco_counts) - min(eco_counts)
score -= eco_diff * 500

# ツインバランス
twin_diff = max(twin_counts) - min(twin_counts)
score -= twin_diff * 800
if twin_diff > 2: score -= (twin_diff - 2) * 1,500

# フロア人数制約
for each floor:
    excess = max(0, hk_count_on_floor - 3)
    score -= excess * 800

return score
```

**_renumber_hks メソッド（番号再割当）**

```
ソート基準:
1. has_bath（True が先）
2. 最小割当フロア番号（昇順）

新しいID: 1, 2, 3, ... （連番）
旧IDから新IDへのマッピング作成
割り当て結果を新IDで再構築
```
