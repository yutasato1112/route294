## 5. デプロイ手順

### 5.1 開発環境構築

```bash
# 1. リポジトリクローン
git clone <repository_url>
cd route294

# 2. Python依存パッケージインストール
pip install -r requirements.txt

# 3. データベースマイグレーション
cd route
python manage.py migrate

# 4. スーパーユーザー作成
python manage.py createsuperuser

# 5. 開発サーバー起動
python manage.py runserver

# アクセス: http://localhost:8000/
```

### 5.2 本番環境デプロイ

```bash
# 1. 環境変数設定
export DJANGO_SECRET_KEY='ランダム文字列'
export DJANGO_DEBUG=False
export DJANGO_ALLOWED_HOSTS='example.com'
export DB_NAME='route294'
export DB_USER='dbuser'
export DB_PASSWORD='dbpassword'
export DB_HOST='localhost'

# 2. 依存パッケージインストール
pip install -r requirements.txt
pip install gunicorn psycopg2-binary

# 3. 静的ファイル収集
python manage.py collectstatic --noinput

# 4. データベースマイグレーション
python manage.py migrate

# 5. Gunicorn起動
gunicorn route.wsgi:application \
    --bind 0.0.0.0:8000 \
    --workers 3 \
    --timeout 120

# 6. Nginx設定（別途）
```

### 5.3 Nginx設定例

```nginx
server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    location /static/ {
        alias /path/to/route294/route/static/;
        expires 30d;
    }

    location /media/ {
        alias /path/to/route294/route/media/;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

## 6. ログ管理

### 6.1 ログファイル仕様

| 項目 | 仕様 |
|------|------|
| 出力先 | logs/log_YYYYMMDD.log |
| ローテーション | 日次（日付ベースのファイル名） |
| フォーマット | `%(asctime)s [%(levelname)s] %(message)s` |
| レベル | INFO, WARNING, ERROR |
| 最大保持件数 | 管理画面表示: 300件 |
| 管理方法 | 管理者パネルから閲覧・削除 |

## 7. セキュリティ設定

### 7.1 本番環境Django設定

```python
# route/route/settings.py （本番用変更点）

import os

# セキュリティ基本
DEBUG = False
SECRET_KEY = os.environ['DJANGO_SECRET_KEY']
ALLOWED_HOSTS = os.environ.get('DJANGO_ALLOWED_HOSTS', 'localhost').split(',')

# HTTPS強制

SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True

# セキュリティヘッダ
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# データベース
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.environ.get('DB_NAME', 'route294'),
        'USER': os.environ.get('DB_USER', 'route294'),
        'PASSWORD': os.environ.get('DB_PASSWORD'),
        'HOST': os.environ.get('DB_HOST', 'localhost'),
```

### 8.2 定期メンテナンス

| 作業 | 頻度 | コマンド/手順 |
|------|------|-------------|
| ログ確認 | 日次 | 管理者パネル → ログ管理 |
| データベースバックアップ | 日次 | `cp db.sqlite3 backup/` |
| セッションクリーンアップ | 週次 | `python manage.py clearsessions` |

| パッケージ更新確認 | 月次 | `pip list --outdated` |
| セキュリティパッチ | 随時 | `pip install --upgrade django` |

### 8.3 障害対応

| 障害パターン | 症状 | 対処 |
|-------------|------|------|
| サーバー停止 | 画面表示不可 | `python manage.py runserver` で再起動 |
| DB ロック | "database is locked" | プロセス再起動、同時接続確認 |
| セッション消失 | データが初期化 | JSONバックアップからリストア |
| CSV破損 | 画面エラー | 管理者パネルからCSV再作成 |
| ディスク容量不足 | ログ書き込み失敗 | 古いログ・メディアファイル削除 |
| メール送信失敗 | レポート未送信 | email.json設定確認、SMTP接続確認 |
