## 1. システム起動・停止

### 1.1 起動手順

```bash
# 1. プロジェクトディレクトリへ移動
cd /.../route

# 2. 開発サーバー起動
python manage.py runserver

# 3. バックグラウンド起動（任意）
python manage.py runserver &

# 4. ポート指定起動（任意）
python manage.py runserver 0.0.0.0:8000
```

**起動確認**:
- ブラウザで http://localhost:8000/ にアクセス
- メイン画面（部屋グリッド）が表示されることを確認

### 1.2 停止手順

```bash
# フォアグラウンド起動の場合
Ctrl + C

# バックグラウンド起動の場合
kill $(lsof -ti:8000)
```

## 3. 管理者運用手順

### 3.1 管理者パネルへのアクセス

```
1. http://localhost:8000/login/ にアクセス
2. スタッフ権限のあるユーザー名・パスワードでログイン
3. 管理者パネル（/administrator/）が表示される
```

### 3.2 ユーザー管理（スーパーユーザーのみ）

#### ユーザー作成

|------|------|
| 1 | 管理者パネル → ユーザー管理セクション |

| 2 | 確認ダイアログで「削除」を選択 |

## 4. トラブルシューティング

### 4.1 ログインできない

| 原因 | 対処 |
|------|------|
| パスワード忘れ | Django shellでリセット: `user.set_password('new_password')` |
| スタッフ権限なし | Django shellで設定: `user.is_staff = True; user.save()` |
| アカウント未作成 | `python manage.py createsuperuser` で作成 |

```bash
# パスワードリセット手順
cd /Users/yutasato/work/rootinn/route294/route
python manage.py shell
>>> from django.contrib.auth.models import User
>>> user = User.objects.get(username='ユーザー名')
>>> user.set_password('新しいパスワード')
>>> user.is_staff = True  # 必要に応じて
>>> user.save()
>>> exit()
```

### 4.2 サーバーが起動しない

| 原因 | 対処 |
|------|------|
| ポート使用中 | `lsof -i:8000` で確認、プロセスを終了 |
| 依存パッケージ不足 | `pip install -r ../requirements.txt` |
| マイグレーション未実行 | `python manage.py migrate` |
| ディレクトリ誤り | `cd /Users/yutasato/work/rootinn/route294/route` |

### 4.3 オートパイロットが失敗する

| 原因 | 対処 |
|------|------|
| クオータ不一致 | 合計クオータと対象部屋数を一致させる |
| ツインクオータ超過 | ツインクオータ合計をツイン部屋数以下に設定 |
| 部屋未選択 | メイン画面で部屋にハウスキーパーIDを入力 |

### 4.4 印刷プレビューが表示されない

| 原因 | 対処 |
|------|------|
| 部屋未割当 | メイン画面で部屋を割り当ててから実行 |
| CSV破損 | 管理者パネルでCSVファイルを確認・修正 |
| 翻訳エラー | 翻訳APIが利用不可の場合、日本語で出力される |

### 4.5 メール送信失敗

| 原因 | 対処 |
|------|------|
| email.json 未設定 | static/email.json を作成 |
| アプリパスワード誤り | Googleアカウントでアプリパスワードを再生成 |
| 2段階認証未設定 | Googleアカウントで2段階認証を有効化 |

```json
// static/email.json の正しい形式
{
    "address": "your-email@gmail.com",
    "password": "16桁のアプリパスワード",
    "developer_address": "受信先メールアドレス"
}
```

### 4.6 セッションデータが消えた

| 原因 | 対処 |
|------|------|
| サーバー再起動 | JSON保存機能で事前にバックアップ |
| セッションタイムアウト | JSON保存機能で定期バックアップ推奨 |
| ブラウザCookie削除 | JSON保存機能からリストア |

**予防策**: 作業中は定期的にJSON保存を実行すること。

### 4.7 データベースロックエラー

```
エラー: "database is locked"
```

| 原因 | 対処 |
|------|------|
| 複数プロセスのアクセス | 同時にrunserverを複数起動していないか確認 |
| 管理コマンド実行中 | shell等のコマンドが実行中でないか確認 |
| 根本対策 | 本番環境ではPostgreSQLを使用 |

## 6. バージョンアップ手順

### 6.1 通常アップデート

```bash
# 1. 現在のデータをバックアップ
cp route/db.sqlite3 route/db.sqlite3.backup
cp -r route/static/csv route/static/csv.backup

# 2. ソースコード更新
git pull origin main

# 3. 依存パッケージ更新（必要な場合）
pip install -r requirements.txt

# 4. マイグレーション実行（必要な場合）
python manage.py migrate

# 5. サーバー再起動
# （開発サーバーは自動リロードあり）
```

### 6.2 ロールバック手順

```bash
# 1. サーバー停止
Ctrl + C

# 2. ソースコード復元
git checkout <前のコミットハッシュ>

# 3. データベース復元（必要な場合）
cp route/db.sqlite3.backup route/db.sqlite3

# 4. CSVデータ復元（必要な場合）
cp -r route/static/csv.backup/* route/static/csv/

# 5. サーバー再起動
python manage.py runserver
```
